***Development Approach***

The project will follow **Kanban**, allowing for flexibility, continuous feedback, and iterative improvement. Kanban was chosen over Scrum for a few practical reasons:
- Greater flexibility: Rather than working in fixed sprints, I can plan and pick up work in a continuous flow making it easier to adapt when priorities change or new information comes in.
- Reduced overhead: With fewer required meetings and a lighter planning process, more time can be spent building the product rather than managing it.
    
- Better fit for small teams: Additionally, Kanban often better suits small teams that do not have a rigid team structure. Team members can contribute where they're needed most, based on capacity and what the project demands at any given time.
    

The project will move through five main **Phases**:

- **Discovery**: Define user needs, research existing solutions and find gaps in current offerings
- **Design**: Translate those findings into wireframes, a database schema, and flow diagrams. 
- **Development**: Incrementally start building  authentication, profiles, search, and rating system. Features will be added incrementally as I progress.
- **Testing**: Conduct unit testing, usability testing, and bug fixing.
- **Deployment**: Host the app online and finalise documentation
